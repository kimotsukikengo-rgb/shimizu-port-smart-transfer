1. システム概要・全体像

プロジェクト概要

清水港に寄港するクルーズ船の乗客（主に外国人観光客）向けに、タクシーチャーター観光ツアーの予約・販売・配車・運行管理を行うWebシステムです。

現状の課題: 紙ベースの受注管理、手書きの運行指示書、Excelでの集計 → これらを統合的なWebシステムで効率化します。

システム全体フロー

業務の時系列フロー

事前準備（前日〜数日前）

寄港情報登録 → 船名・入出港時刻・W番号枠を設定 → 事前予約受付（URL A）

当日（対面販売）— 受注〜決済

営業がURL Bで受注入力 → 受付番号 自動発行 → 決済デスクで受付番号確認 → 内容確認（修正可） → mPOS/全旅/現金/Web決済 → W番号入力（車両ごと）

当日（対面販売）— 配車〜出発

配車担当が車両・乗務員引当 → 運行指示書PDF生成（車両ごと） → 乗務員にLINE/メール送信

運行中〜完了

乗務員がツアー実施 → 乗務員がスマホで運行完了報告 → 管理画面で実績確認・集計 → 行政レポート出力

3. データモデル・エンティティ設計

重要: 番号体系

本システムには2種類の番号が存在し、混同厳禁です。

番号名称発行タイミング形式用途
受付番号Reception Number受注入力時に自動採番R-YYYYMMDD-NNNシステム内の一意識別子。全工程で使用
W番号Waiting Number決済完了時に物理カードから読取り。1台=1W番号（3台口なら3枚使用）W-NNN顧客が乗車待ち時に使う物理カードの番号。寄港日ごとにリセット。車両単位で発行（配車テーブルに紐づく）

注意: W番号は寄港情報登録で事前に枠を設定（チャーター・ワンウェイ共通連番）。物理カードに印刷済みの連番を使用。

重要（4/1打合せ決定）: W番号は車両単位で発行されます。例: 3台口の受注はW-005, W-006, W-007の3つのW番号を使用し、配車（dispatches）テーブルに紐づきます。受注（orders）テーブルではなく配車テーブルで管理します。

ステータス遷移（4/1打合せ決定事項）

ステータス統一方針

画面表示上のステータスは「帰着待ち」と「運行完了」の2値に統一する。内部的には以下のステータスチェーンで管理。

受注完了→ 決済待ち→ 決済完了→ 配車待ち→ 帰着待ち→ 運行完了

管理画面・URL Bともに表示は「帰着待ち」「運行完了」のみ。フィルタリング用に内部ステータスを保持。

キャンセル: 任意のステータスからキャンセル可。キャンセル理由（プルダウン＋自由記入）を記録。売上0円計上。キャンセル済みW番号は一覧上で取消線表示。

5. URL B: スタッフ専用 対面販売端末

画面遷移フロー（14画面、5ロール）

ロール別フロー

受注入力（営業）: ログイン → 役割選択 → 受注入力（コース・車両車種ごと人数・顧客情報・ドロップオフ・特記事項・オプション項目・営業担当名） → 受注確認 → 受付番号発行

決済: ログイン → 役割選択 → 受付番号検索（or 決済待ちリスト） → 内容確認（修正可） → 決済方法選択（mPOS/全旅/現金/Web） → W番号入力（車両ごと） → 決済担当名選択 → 決済完了

配車入力: ログイン → 役割選択 → 受付番号検索（or 配車待ちリスト） → 車両ごとに配車情報入力（タクシー会社・乗務員・無線番号3桁・車種・メール） → 出発時刻入力 → 運行指示書プレビュー → 印刷/メール/LINE送信 → 配車完了

受注修正: ログイン → 役割選択 → 受付番号検索（or 修正可能案件リスト） → 受注内容修正 → 再登録

乗務員: メールリンクから直接アクセス（ログイン不要） → 運行完了報告（行き先・帰着時刻・延長/返金・決済料金/手数料・原価5項目・顧客満足度・メモ） → 送信

運用上の特殊フロー

W番号カードの物理運用:

  1. 寄港情報登録で W番号の枠を事前設定（例: W-001〜W-060）
  2. 物理カード（プラスチック製、番号印刷済み）を準備
  3. 決済完了時に、スタッフが物理カードの番号をシステムに手入力
  4. カードに受付番号を手書きし、顧客に渡す
  5. 顧客はこのカードを持ってタクシー乗り場で待機
  6. 配車担当がカードの番号で受注を特定し、車両を割り当て

→ W番号は「物理カードとの紐付け」がキモ。バーコード/QR化は将来検討。

7. 寄港情報登録・業務分析

なぜ「寄港情報登録」が必要か

対面販売を一斉に開始する時点で、各営業が「クルーズ船名」をバラバラに入力するのはNG。

→ 事前に管理者が寄港情報を登録し、営業スタッフの受注入力画面に自動反映させる。

これにより、船名・日付・天気情報が全スタッフで統一され、集計・分析の精度が保証されます。

10. 法令対応・旅行業法

旅行業法上の位置づけ

本サービスは「募集型企画旅行」（パッケージツアー）に該当します。旅行業法に基づき以下が必要です。

  • 旅行条件書の事前交付: 予約前に旅行条件書を提示し、顧客の明示的な同意を取得。電磁的方法（Web画面上）での交付可。
  • 取消料規定の明示:
    • 7日前まで: 無料
    • 3〜6日前: 旅行代金の30%
    • 1〜2日前: 旅行代金の50%
    • 当日: 旅行代金の100%
  • 旅行業務取扱管理者の明示: サイト上に記載必須
  • 運行指示書の保管: 帳票として一定期間保管義務
  • 行政報告: 国土交通省への定期報告（CSV/PDF出力機能で対応）
